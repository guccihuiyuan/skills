# Keyblade Pro Workflow

## 技能概述

这是一个**工作流编排器**，用于协调多个技能完成AI代码编写任务。核心原则是**不要一上来就写代码**，而是先进行需求分析和计划制定。

**核心特性：**
- 支持技能自动检测：在执行每个步骤前检测依赖技能是否存在
- 自动跳过缺失技能：若某个技能不存在，自动跳过该环节并继续后续流程
- 灵活的流程控制：保证工作流的连续性和可用性

## 何时触发

**必须立即使用此技能的情况：**
- 用户要求"编写代码"、"生成代码"、"实现功能"
- 用户要求"修改代码"、"重构代码"、"优化代码"
- 用户要求"创建组件"、"开发页面"、"实现接口"
- 用户提出新的功能需求或 bug 修复需求

## 工作流流程

### 步骤 1：需求分析 (brainstorming)

**首先检测并调用 `brainstorming` 技能**，完成以下工作：
- 深入理解用户需求
- 分析功能边界和技术选型
- 探索可能的实现方案
- 识别潜在风险和依赖

**输出要求：**
- 明确的需求文档
- 技术方案概述
- 关键设计决策

**技能检测与跳过逻辑：**
- 在执行前检测 `brainstorming` 技能是否存在
- 若技能不存在，自动跳过此步骤，直接进入计划制定阶段
- 跳过前向用户说明："未检测到 brainstorming 技能，将跳过需求分析环节"

### 步骤 2：计划制定 (writing-plans)

**需求分析完成后，检测并调用 `writing-plans` 技能**，完成以下工作：
- 创建详细的开发计划
- 分解任务到可执行的单元
- 定义任务优先级和依赖关系
- 预估时间和资源需求

**输出要求：**
- 完整的任务列表
- 任务优先级排序
- 时间估算

**技能检测与跳过逻辑：**
- 在执行前检测 `writing-plans` 技能是否存在
- 若技能不存在，自动跳过此步骤，直接进入代码编写阶段
- 跳过前向用户说明："未检测到 writing-plans 技能，将跳过计划制定环节"

### 步骤 3：代码编写 (keyblade-pro-code-rules)

**计划制定完成后，检测并调用 `keyblade-pro-code-rules` 技能**，完成以下工作：
- 按照项目规范编写代码
- 生成类型定义、枚举、请求函数等
- 创建组件、页面、接口等
- 遵循 Vue3 + TypeScript + Arco Design 规范

**输出要求：**
- 完整的代码文件
- 符合规范的代码结构

**技能检测与跳过逻辑：**
- 在执行前检测 `keyblade-pro-code-rules` 技能是否存在
- 若技能不存在，自动跳过此步骤，直接进入代码审查阶段
- 跳过前向用户说明："未检测到 keyblade-pro-code-rules 技能，将跳过规范检查环节"

### 步骤 4：代码审查与验证

代码编写完成后进行：
- 代码规范检查
- 类型安全检查
- 运行 lint 和 typecheck

## 工作流约束

### 必须遵守的规则
1. **按顺序执行**：按照 brainstorming → writing-plans → keyblade-pro-code-rules 的顺序尝试执行
2. **不要一上来就写代码**：在有条件的情况下，优先进行需求分析和计划制定
3. **技能检测优先**：每个步骤执行前必须先检测依赖技能是否存在
4. **自动跳过机制**：若依赖技能不存在，自动跳过该环节并继续后续流程
5. **跳过通知**：跳过环节前必须向用户说明原因
6. **输出驱动**：每个步骤必须产生明确的输出成果

### 步骤间的依赖关系

```
brainstorming ──(检测)──→ writing-plans ──(检测)──→ keyblade-pro-code-rules
       │                       │                        │
       ▼                       ▼                        ▼
  需求分析(可选)          计划制定(可选)            代码编写(可选)
       │                       │                        │
       └───────────────────────┴────────────────────────┘
                              │
                              ▼
                       代码审查与验证
```

### 技能检测流程

```
开始工作流
    │
    ▼
检测 brainstorming 技能
    │
    ├── 存在 → 执行需求分析 → 进入下一步
    └── 不存在 → 跳过并提示用户 → 进入下一步
            │
            ▼
检测 writing-plans 技能
    │
    ├── 存在 → 执行计划制定 → 进入下一步
    └── 不存在 → 跳过并提示用户 → 进入下一步
            │
            ▼
检测 keyblade-pro-code-rules 技能
    │
    ├── 存在 → 执行代码编写 → 进入下一步
    └── 不存在 → 跳过并提示用户 → 进入下一步
            │
            ▼
执行代码审查与验证
    │
    ▼
结束工作流
```

## 核心依赖技能

| 技能名称 | 用途 | 调用时机 | 检测与跳过 |
|---------|------|---------|-----------|
| `brainstorming` | 需求分析和方案设计 | 工作流第一步 | 不存在时跳过，直接进入计划制定 |
| `writing-plans` | 创建开发计划和任务列表 | 需求分析完成后 | 不存在时跳过，直接进入代码编写 |
| `keyblade-pro-code-rules` | 代码编写和规范检查 | 计划制定完成后 | 不存在时跳过，直接进入代码审查 |

## Vue3 + TypeScript + Arco Design 项目特殊处理

### 表格页开发流程
1. 需求分析：明确表格功能需求
2. 计划制定：分解任务（类型定义、请求函数、组件开发）
3. 代码编写：使用 kb-pro-table、useTable、useForm

### 组件开发流程
1. 需求分析：明确组件功能和接口
2. 计划制定：分解任务（props定义、事件处理、样式开发）
3. 代码编写：使用 arco design + unocss

## 输出规范

### 每个步骤必须输出
- **brainstorming**：需求文档、技术方案
- **writing-plans**：任务列表、优先级排序
- **keyblade-pro-code-rules**：代码文件、测试文件

### 最终交付
- 完整的代码文件
- 代码审查报告
- 功能验证结果

## 错误处理

### 需要停止并询问用户的情况
- 需求不明确或存在歧义
- 技术栈无法识别
- 规范冲突无法解决
- 需要用户确认的设计决策

### 工作流中断处理
- 记录当前步骤和进度
- 保存已完成的成果
- 向用户说明中断原因
- 提供恢复建议

## 最佳实践

1. **遵循流程**：严格按照 brainstorming → writing-plans → keyblade-pro-code-rules 的顺序执行
2. **不要急于编码**：充分的需求分析和计划制定是高质量代码的基础
3. **文档驱动**：每个步骤都要有明确的输出文档
4. **规范优先**：代码编写必须遵循项目编码规范
5. **验证完整性**：代码完成后必须进行审查和测试
